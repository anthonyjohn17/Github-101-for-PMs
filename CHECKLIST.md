# Your First 3 Weeks on GitHub

Use **[Github-101.md](Github-101.md)** anytime you want the full picture on a command, error, or workflow (staging, branches, PRs, undo, stash, auth, and advanced topics). The checklist below is your shipping plan; the guide is your reference manual.

---

## Day 1: Profile Setup
- [ ] Create GitHub account (use your real name as username)
- [ ] Add bio: "PM building [specific thing]. Shipping [type of projects]."
- [ ] Add location, LinkedIn link, portfolio link
- [ ] Create profile README repo (repo name = your username)
- [ ] Write profile README using `templates/PROFILE-README.md`
- [ ] Install Cursor (cursor.sh) or Claude Code (docs.claude.com/claude-code)
- [ ] Connect GitHub account to your coding tool

## Day 2: First Repository
- [ ] Pick a project idea (start with a problem you actually have)
- [ ] Create repo using `prompts/create-repo.md`
- [ ] Build project using `prompts/build-project.md`
- [ ] Commit and push using `prompts/commit-and-push.md`
- [ ] Write README using `prompts/write-readme.md`
- [ ] Verify it's live: visit github.com/yourusername/project-name

## Week 2: Flagship Project
- [ ] Identify your biggest PM pain point
- [ ] Create a new repo for your flagship project
- [ ] Build it across 3-5 sessions (each session = a commit)
- [ ] Write comprehensive README with Tradeoffs section
- [ ] Add screenshots or demo GIF
- [ ] Pin this repo on your GitHub profile

## Week 3: Polish and Publish
- [ ] Review all READMEs — would a hiring manager understand your thinking?
- [ ] Add screenshots/GIFs to any project that's missing them
- [ ] Update profile README with your new projects
- [ ] Pin your best 2-3 repos
- [ ] Add GitHub link to your LinkedIn and resume
- [ ] Share your flagship project on LinkedIn with a post about what you built and why

## Ongoing
- [ ] Build or improve one project per week
- [ ] Keep commit messages clear and descriptive
- [ ] Fork and contribute to tools you actually use
- [ ] Store your prompt libraries, templates, and evaluation rubrics as repos

## Optional: deepen Git and GitHub (any week)

- [ ] Skim the **table of contents** at the top of [Github-101.md](Github-101.md) so you know where topics live
- [ ] Read [Quick Start](Github-101.md#-quick-start-5-steps), [Staging](Github-101.md#-staging-files), [Committing](Github-101.md#-committing-changes), and [Pushing](Github-101.md#-pushing-code) before your second week of commits
- [ ] When you open a PR, read [Pull Requests](Github-101.md#-pull-requests) and [Merge Conflicts](Github-101.md#-merge-conflicts)
- [ ] Bookmark [Common Errors & Fixes](Github-101.md#-common-errors--fixes) and [Undoing Mistakes](Github-101.md#-undoing-mistakes) for when something goes wrong
